# EBU6304-TA-System-Group11-国际学院助教招聘系统 

## 📌 项目概览
本项目为 BUPT 国际学院开发的助教（TA）招聘管理系统，旨在优化 TA 岗位发布、申请流程与状态追踪效率。
- **开发模式**
：敏捷开发 (Agile Methods)
- **技术栈**
：Java 17（无数据库，纯文本文件持久化）
- **验收节点**
：
  -
 第一阶段 (Phase 1) - 2026年3月22日（需求梳理+原型设计+分支规范落地）
  -
 后续阶段（待确认）

## 🌿 Git 分支管理规范 
### 1. 分支命名规则（核心修正：适配个人贡献验收）
| 分支类型       | 命名格式                  | 用途说明                                                                 |
|----------------|---------------------------|--------------------------------------------------------------------------|
| `main`         | -                         | 仅存放验收版本的稳定代码，**严禁直接提交/推送**，仅通过 PR 合并            |
| `develop`      | -                         | 小组公共开发汇聚分支，所有成员完成功能后合并至此                          |
| `dev/姓名全拼` | 例：`dev/Zhaolun-Li`      | 个人开发分支（Phase 1 验收专用，替代 feature 分支，便于助教核查个人贡献）|
| `feature/功能` | 例：`feature/login`       | （Phase 2 开发用）按功能模块创建的协作分支                               |

### 2. 协作流程（适配 Phase 1 验收，简化操作）
#### （1）Phase 1 个人贡献阶段（3月22日前）
- 步骤1：登录个人 GitHub 账号，从 `develop` 分支创建专属个人分支 `dev/姓名全拼`
-
 步骤2：在个人分支内上传调研材料/原型设计/需求文档（无需合并到 develop）
-
 步骤3：确保个人分支有清晰的提交记录（Commit），便于助教验收

#### （2）Phase 2 开发协作阶段（验收后）
- 开发前：切换到 `develop` 分支，执行 `Pull`
 拉取最新代码
- 开发中：从 `develop` 创建 `feature/功能名`
 分支开发具体功能
- 开发后：Push 分支到 GitHub，发起 PR 合并回 `develop`
，由组长审核

### ⚠️ 核心约束（技术红线）
1. **数据存储**：严禁使用 MySQL/SQLite 等数据库，所有数据统一存储在 `data/` 目录下的 `.json`/`.txt`
 文件中
2. **代码结构**：业务逻辑代码必须放在 `src/` 目录下，资源文件（配置/数据）放在 `data/`
3. **权限规范**：仅组长可操作 `main` 分支，所有成员可读写 `develop`
 分支，个人分支仅本人操作
